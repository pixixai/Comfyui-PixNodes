# Pix_ListFromLoop (从循环创建列表) 节点说明

## 1. 节点功能概览

`Pix_ListFromLoop` 是专为循环工作流设计的状态保持节点，属于 `PixNodes` 系列。在 ComfyUI 的循环（Loop）运行过程中，该节点负责“记忆”并“累加”每一轮循环产生的输出，最终将其汇聚成一个完整的列表。

## 2. 输入参数解释

| 参数名称 | 类型 | 是否必填 | 描述 |
| --- | --- | --- | --- |
| **item** | * (任意) | 是 | 当前循环轮次产生的新数据项。 |
| **list_in** | * (列表) | 否 | 上一轮循环传回的已累积列表。在第一轮循环时通常保持为空。 |

> 工作原理：它会创建一个新列表，将 list_in 中的既有内容复制进去，再把当前的 item 追加到末尾。
> 

## 3. 输出结果描述

| 索引 | 名称 | 类型 | 描述 |
| --- | --- | --- | --- |
| **0** | list | * (列表) | 包含当前项及之前所有累积项的完整列表。 |
| **1** | count | INT | 当前列表中数据项的总数。 |

## 4. 典型工作流场景

- **批量图像生成**: 在循环中每步生成一张图，通过此节点收集，最后连接到 `Pix_ListToImageBatch` 合并为批次。
- **参数扫描**: 在循环中尝试不同的 Prompt 或种子值，利用此节点记录下所有生成的文本或数值结果，以便后续进行对比分析。
- **迭代状态保持**: 配合 ComfyUI 的循环节点（如 Loop Controller 等），将当前输出作为下一轮的输入参考。
