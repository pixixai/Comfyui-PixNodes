# List from Loop (从循环获取列表)

## 节点功能概览

`List from Loop` 是循环工作流（For Loop / While Loop）中的核心收纳工具。它的主要功能是在每一轮迭代过程中“收集”产生的数据项，并将其累加到一个持久化的 Python 列表中。

该节点针对 ComfyUI 的图像批次（Batch）进行了特殊优化，支持自动拆解合并，是构建复杂循环任务（如批量生成、图像序列处理）的必备组件。

## 输入参数解释

### 必须输入 (Required)

- **list_in** ():
    - **描述**: 接收来自上一轮循环的列表对象。在标准循环流中，此端口应连接至 `Loop Start` 节点的 `value` 端口或本节点的输出闭环。
- **item** ():
    - **描述**: 接收当前这一圈产生的新数据。支持任何类型（图像、潜变量、字符串等）。
- **split_items** (`BOOLEAN`):
    - **建议值**: `True` (默认)。
    - **描述**:
        - **开启 (Split)**: 如果输入的 `item` 是一个包含多张图的图像批次（Batch），则会将其拆解为多张独立的单图依次存入列表。
        - **关闭 (Keep)**: 将 `item` 作为一个整体存入列表。

### 可选输入 (Optional)

- **manual_reset_debug** (`BOOLEAN`):
    - **描述**: 调试专用开关。勾选后，节点将强行丢弃之前的累加记录，从空列表重新开始。仅建议在工作流调试出现“数据残留”时使用。

## 输出结果描述

- **0 (list)**: 累加后的完整 Python 列表对象。通常用于传递给下一轮循环或最终的列表处理节点（如 `ListToImageBatch`）。
- **1 (count)**: 整数。返回当前列表中已收集的元素总数。

## 典型工作流应用场景

1. **动画序列收集**: 在循环中逐帧生成图片，最后一次性合并为视频序列。
2. **批量提示词存储**: 收集每一轮循环生成的动态文本描述，用于后续的统一保存或对比。
3. **数据过滤与汇总**: 配合逻辑节点，在循环中只收集符合特定条件的数据。
