# 从视频列表获取视频

此节点用于从给定的**视频对象列表**中，根据索引选择并提取单个视频。它专为处理 ComfyUI 的 `VIDEO` 类型数据流而设计。

## 功能特性

- **列表处理**：能够接收并扁平化处理来自上游的视频列表（支持嵌套列表）。
- **循环索引**：内置取模逻辑 (`index % total`)，即使索引超出列表范围也能自动循环回到开头，非常适合批量处理或循环播放场景。
- **纯逻辑处理**：仅负责数据的选择与分发，不进行解码或预览，运行极其高效。

## 输入参数 (Inputs)

| 参数名 | 类型 | 描述 |
| --- | --- | --- |
| **video_list** | `VIDEO` (List) | 连接包含 `VIDEO` 对象的列表（通常来自 `GetVideoPathList` 或其他批量加载节点）。 |
| **index** | `INT` | 要获取的视频在列表中的索引位置（从 0 开始）。

支持自动循环：例如列表有 3 个视频，输入索引 3 会自动选取第 0 个视频。 |

## 输出 (Outputs)

| 索引 | 名称 | 类型 | 描述 |
| --- | --- | --- | --- |
| **0** | **video** | `VIDEO` | 选中的单个视频对象。可直接连接到 `Get Video Components`, `Save Video` 等官方节点。 |
| **1** | **index** | `INT` | 当前实际使用的有效索引（经过取模计算后的值）。 |
| **2** | **total** | `INT` | 输入列表中的有效视频总数。 |

## 使用示例

1. **从列表中按序提取**：
    - 上游：使用 `Pix_GetVideoPathList` 加载多个视频。
    - 本节点：连接 `video_list`，设置 `index`（或连接 `Primitive` 节点实现自增）。
    - 下游：连接 `Save Video` 或 `Get Video Components` 进行处理。

**注意**：此节点仅处理 `VIDEO` 对象类型，不处理纯文本路径。如需从文本路径加载，请先使用加载器节点。
