# 数据判空节点 (Any Data Is Empty)

## 节点功能概览

`AnyDataIsEmpty` 是一个多功能的逻辑校验节点，专门用于检测进入节点的各种数据流是否为“空”。它通过底层重写 `VALIDATE_INPUTS` 方法，实现了对 ComfyUI 所有数据类型的兼容（通配符输入），解决了常规节点无法直接连接不同类型插槽进行逻辑判断的痛点。

## 输入参数解释

| 参数名称 | 类型 | 默认值 | 描述与建议 |
| --- | --- | --- | --- |
| **invert** | BOOLEAN | False | **反转逻辑开关**。默认情况下，若数据为空则输出 `True`。开启此开关后，逻辑反转：数据为空输出 `False`，数据有效则输出 `True`。常用于“如果有数据则继续执行”的场景。 |
| **any_data** | * (通配符) | None | **待检查的数据**。可以连接图片、遮罩、Latent、文本字符串、数字、JSON 字典或列表。 |

## 输出结果描述

- **is_empty**: 输出一个布尔值（BOOLEAN）。该值可直接连接至逻辑门（Logic Gate）或条件分支节点，用于控制工作流的走向。

## 典型工作流应用场景

1. **自动化条件分支**：在视频处理工作流中，检查前一阶段是否生成了遮罩（Mask），如果没有，则跳过重绘步骤。
2. **API 健壮性校验**：当从外部 JSON 获取数据时，检查字段是否为空，防止下游节点报错导致工作流中断。
3. **文本提示词判空**：检查动态生成的提示词是否为空字符串，如果是，则切换到默认提示词。

## 技术特性

- **全类型支持**：自动识别并处理 `torch.Tensor` (图像/遮罩)、`dict` (Latent/JSON)、`list`、`str` (文本) 等。
- **类型安全**：对于无法判空的类型（如单一数字），默认视为非空。
