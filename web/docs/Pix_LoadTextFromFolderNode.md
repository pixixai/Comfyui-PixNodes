# 从文件夹加载文本

## 1. 节点功能概览

`Pix_LoadTextFromFolder` 节点用于从指定本地目录中批量扫描 `.txt` 文件，并根据索引（Index）精准提取单个文件的内容。该节点采用了**自然排序算法**（Natural Sorting），确保如 `1.txt`, `2.txt`, `10.txt` 这样的文件名能按数字逻辑而非单纯字符逻辑排序。

## 2. 输入参数解释

| 参数名称 | 类型 | 建议值 | 描述 |
| --- | --- | --- | --- |
| **directory** | STRING | `/mypath/prompts` | 存放文本文件的文件夹绝对路径。 |
| **start_index** | INT | `0` | 要加载的文件索引。0 代表排序后的第一个文件。 |

> 特性：节点支持 lazy 模式，且在文件索引越界时会自动箝位（Clamp）到最后一个有效文件，防止工作流因报错中断。
> 

## 3. 输出结果描述

| 索引 | 名称 | 类型 | 描述 |
| --- | --- | --- | --- |
| **0** | content | STRING | 被选中文件的完整文本内容。 |
| **1** | file_name | STRING | 不带后缀的文件名，方便后续作为保存图片的文件名。 |
| **2** | index | INT | 当前实际加载的文件索引。 |
| **3** | total_files | INT | 文件夹中符合条件的文本文件总数。 |

## 4. 典型工作流场景

- **批量 Prompt 渲染**: 将成百上千个提示词分别存为 txt 文件，配合循环节点累加索引，实现自动化的批量跑图。
- **动态元数据读取**: 读取与图像对应的描述文件，并将 `file_name` 传给 `Save Image` 节点，实现文件名的自动同步。
